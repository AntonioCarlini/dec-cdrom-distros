         1 ALLMÄN BESKRIVNING

              DIGITAL[TM] Office Server for OpenVMS[TM] V4.0A bygger
              på den bevisade styrkan hos både ALL-IN-1[TM] och
              MailWorks[TM], som tillhandahåller robusta och skalbara
              meddelandetjänster till en stort antal postklienter på
              en rad olika serverplattformar, och som kommunicerar med
              flera meddelandeposttjänster. Till de främsta funktionerna
              räknas:

              o  Den nya Klienten Administrator flyttar den dagliga
                 administrationen till ett nytt grafiskt användar-
                 gränssnitt (GUI) på Windows NT[TM] V4.0 och Windows
                 95[R].

              o  Alternativet "Endast arkiv" som avaktiverar lokal post
                 och fjärrpost och omdefinierar Office Server som en
                 skalbar informationsbehållare.

              o  Distribuerat arkiv, där användare på lokala system
                 kan nå alla typer av meddelanden och dokument på
                 fjärrsystem som om de fanns lokalt.

              o  Stöd för LDAP (Lightweight Directory Access Protocol)
                 via TCP/IP möjliggör åtkomst till den nya generationen
                 av LDAP-katalogtjänster som blir allt vanligare på
                 Internet och intranät.

              o  Möjlighet att skapa, sända besvara och vidarebefordra
                 post från Internet med personlig åtkomst till Office
                 Server-arkivet med inloggning och lösenordskontroll i
                 samband med den nya DIGITAL Office Server Web Interface
                 V5.0 som installeras på en lokal Internet-server. Även
                 möjlighet att publicera alla typer av dokument för
                 allmän åtkomst finns fortfarande.

              o  Stöd för flera posttjänster:

                 -  X.400 via MAILbus[TM] 400 och DIGITAL X.500 på
                    OpenVMS och UNIX[R] (inklusive smtp/MIME- gateway)

                 -  smtp/MIME via direktstöd för PMDF[R] på flera
                    plattformar utan krav på Message Router

                 -  VAX[TM]-baserad Message Router

        4 ÅTKOMST TILL WEB

              Användare av Internet eller intranät kan besvara,
              vidarebefordra, skapa och sända post medan de använder
              sina egna personliga lådor, mappar, meddelanden och
              dokument, och andra till vilka behörighet har tilldelats,
              med inloggning och lösenordskontroll via DIGITAL Office
              Server Web Interface V5.0 som levereras separat. Med
              hjälp av standardwebbläsare som Netscape Navigator[R]
              och Microsoft Internet Explorer från t ex ett Internet-
              café, kan meddelanden och dokument visas och hämtas till
              PC:n.

              Alla dokument som placeras i en låda med läs- eller
              Internet-åtkomst kan nås från valfri plats på Internet
              (via brandväggar och eventuella tunneling-programvara).
              Det krävs inget speciellt HTML-verktyg (Hypertext Markup
              Language) på servern varken för att skapa eller uppdatera
              dokument för Internet. Ytterligare information finns i SPD
              61.55.01 för V5.0 och 61.55.00 för V4.0.

        6 ALTERNATIVET "ENDAST ARKIV"

              Under installationen av Office Serve V4.0A finns ett
              alternativ som avaktiverar alla lokala och externa
              meddelanden till slutanvändare. Kunder bör välja det här
              alternativet om de vill standardisera på alternerande

              Office Server-arkivet. Dessa klienter kan även använda ett

        7 MEDDELANDE- OCH REGISTERTJÄNSTER

              Office Server kan kommunicera med ett antal meddelan-
              deöverföringsagenter (MTA:er) och katalogtjänstagenter
              (DSA:er).

        7.1 MAILbus 400

              Med X/Open[TM] XAPI- och XDS- gränssnitts-stubs kan
              kunder välja X.400 Alpha-baserade (antingen OpenVMS eller
              DIGITAL UNIX[TM]) MAILbus 400 och nödvändiga DIGITAL X.500
              Directory Service Agents som posttjänst och använda MTA:er
              och DSA:er när och var de vill, med lokal anslutning eller
              fjärranslutning till Office Servers.

              Med Message Router Migration Package V1.0 (QP-042) kan
              Message Router (MR)-kunder övergår från MR till MAILbus
              400 genom att inkludera MAILbus 400 Message Router Gateway
              (XMR) och MAILbus Directory Migration Toolkit (DMT)
              såväl som MAILbus 400 MTA och DSA. Detta paket ombesörjer
              övergång i faser av tusentals användare under en längre
              tidsperiod.

        7.2 PMDF

              Ett annat alternativ är PMDF från Innosoft vilka levererar
              en AlphaServer[TM]-baserad smtp/MIME-posttjänst, med
              direktåtkomst till Office Servers MRIF V2.0 gränssnitt,
              utan behov av Message Router. PMDF kan utnyttja olika
              registertjänster, inklusive sin egen PMDF- X.500/LDAP
              Server, såväl som Network profile i Office Server. PMDF
              stöder inte DDS (Distributed Directory Services) i Message
              Router.

              o  PMDF-X.500/LDAP Server

              o  Netscape Directory Server V3.0

              Adressökning stöds även från TeamLinks, MAPI- drivrutink-

              o  Möjligheten att nå vilken personlig Office Server-mapp

                 till klienten.

        9 FÖRBERETT FÖR ÅR 2000

              Office Server har förberetts för år 2000. Alla datum har
              lagras i OpenVMS 64-bitars format eller NBS-format och det
              stödda datumintervallet är 17:e november 1858 till 31:e
              december 9999.

              År som anges med två siffror blir automatiskt seklet för
              aktuellt systemdatum och lagras så detta. Datumsträngen
              29-april-74 t ex som bearbetas den 5:e mars 1998 lagras
              som 29-apr-1974, och den 5:e mars 2001 som 29-apr-2074.

              Datumsökningar över tusenårsskiftet behandlas korrekt när
              fullständigt datum (och tid) lagras.

        10 DISTRIBUERAT ARKIV

              Distributed Sharing Option är inkluderat i Office Server
              som standard, vilket gör det möjligt för användare på
              fjärrsystem att nå dokument i utsedda lådor och mappar som
              om de vore lokala.

              Allteftersom multimedia-dokument blir större och större
              och distributionslistor längre bidrar möjligheten
              att direkt nå information till att belastningen på
              meddelandesystem minskar. Detta leder i sin tur till
              en prestandaförbättring. Grupper av användare får ett
              e-postmeddelade om att det finns ett dokument i en lokal
              mapp och kan på så sätt välja om de vill hämta det eller
              inte.

              Användare med lämplig behörighet, administratörer och
              systemansvariga kan skapa åtkomstlistor och på så sätt
              försäkra sig om att endast behöriga användare och grupper
              av användare får åtkomst till dokument och meddelanden
              inom vissa lådor och mappar.

        12 ARKIV/ARKIVHANTERING

              Arkivet i Office Server är en behållare för multimedi-
              afiler. Dokument och meddelande av alla typer som HTML,
              Postscript[R], .DOC, .XLS, .PPT, MPP, JPEG, video, audio
              och ASCII kan lagras i arkivet.

              Arkivet är feltolerant eftersom det i händelse av ett
              arkivserverfel eller AIDA-serverfel, vid omstart,
              automatiskt försöker återansluta till MAPI-klienter.
              Sådana klienter inkluderar Outlook, Exchange och Windows
              Messaging.

              Arkivet använder sig av delningsmodellen som innebär
              att ett meddelande som sänds till flera mottagare på
              det lokala systemet inte kopieras till alla mottagare,
              utan använder pekare som gör att alla mottagare läser en
              och samma kopia av meddelandet. Detta medför två fördelar
              framför modellen med en kopia för varje mottagare; bättre
              prestanda och mycket lägre krav på diskutrymme.

              Arkivet påminner om ett vanligt arkivskåp; med lådor med
              dokument i mappar. Mapparna ligger i alfabetisk ordning
              och dokumenten i mapparna ligger i kronologisk ordning.

              Information om varje dokument (som titel, tillkomstdatum,
              dokumentnummer, författare, nyckelord och typ) förvaras
              i arkivet. Det finns även funktioner så att användaren
              kan hantera innehållet i arkivet. Användaren kan skriva
              ut eller ta bort flera dokument åt gången och välja ut
              dokument enligt egna urvalskriterier.

              Ett eller flera dokument åt gången kan flyttas, läggas
              i flera mappar och kopieras. Information såsom titel och
              nyckelord kan ändras. När användaren väljer att lägga
              flera kopior av dokumentet i samma låda sparas endast en
              kopia av dokumentet vilket minskar utrymmesbehovet.

              Dokument som användaren tar bort sparas i mappen
              PAPPERSKORG tills denna töms.

              I arkivet kan användaren komma åt dokument oavsett
              dokumentformat. Som standard stöds alla typer av format,
              även multimedia. Dessutom kan alla X.400 P2-attribut
              lagras i Office Server-arkivet.

                                                                      11

 






              Med funktionen för låsning kan varja användare skydda
              ett dokument medan det redigeras, lokalt eller från en
              fjärrplats. Detta förhindrar att andra användare ändrar
              det tills det låsas upp igen. Under tiden det är låst kan
              andra användare läsa det men inte redigera dokumentet.

              Dokumenthanteringsmodellen gör en klar skillnad mellan
              revideringsbara dokument och dokument i slutligt format.
              Modellen skiljer på formatering och utskrift, vilket
              gör att formatering av dokument kan utföras skilt från
              utskrift.

        12.1 Mail Janitor

              Funktionen Mail Janitor ingår i basprodukten. Den
              systemansvarige kan flytta gamla meddelanden i mapparna
              LÄSTA och SÄNDA till PAPPERSKORGEN. Den systemansvarige
              kan ange högsta antalet dagar som ett meddelande kan
              lagras i mapparna LÄSTA och SÄNDA (standardinställningen
              är 60 dagar) och det högsta antalet meddelanden som får
              tas bort per mapp. Alternativen Profile och Policies
              gäller före standardinställningar för antalet meddelanden
              och den tid som användaren är borta.

        12.2 Checkmail

              Funktionen används för att kontrollera om användaren har
              några olästa eller osända meddelanden i standardlådan
              innan Office Server startas och informerar användaren om
              så är fallet.

        12.3 Gemensamt arkiv

              Användaren kan skapa gemensamma lådor för att ge andra
              användare möjlighet att läsa och redigera gemensamma
              dokument. Lådans ägare har full åtkomstkontroll och
              styr vilka användare som skall komma åt lådan och vilka
              funktioner dessa skall få utföra på dokumenten. När
              användaren skall göra ändringar i ett gemensamt dokument,
              kan han eller hon låsa det så att ingen annan kommer åt
              det under tiden. Den som då försöker redigera dokumentet
              får meddelande om att det används samt om vem som har låst
              det.


        12

 






              Med funktionen för låsning kan användare av Outlook,
              TeamLinks eller VT låsa ett dokument i arkivet innan det
              laddas på användarens arbetsstatione, och visa för alla
              andra användare som har åtkomst till dokumentet att det
              redigeras. Om någon annan försöker redigera dokumentet,
              informeras vederbörande om att det är låst och endast
              kan läsas. Först när låsningen har tagits bort kan andra
              användare få åtkomst till dokumentet.

        12.4 Grupper

              Användare med lämpliga profilinställningar kan definiera
              ett antal användare som en grupp. Flera grupper kan
              definieras och grupperna kan få åtkomstbehörighet till
              gemensamma dokument eller lådor. Medlemmar kan läggas till
              och tas bort från en grupp och deras åtkomstbehörighet
              regleras därvis automatiskt eller tas bort. Användare kan
              även se efter vilka grupper de tillhör.

        13 ELEKTRONISK POST

              Följande information är specifik för Office Server VT-
              klientåtkomst, och beskriver tillgänglig funktionalitet.
              Många av dessa operationer kan nås från TeamLinks och även
              MAPI- tillämpningar. En komplett lista över varje produkts
              funktioner finns i relevant SPD. Meddelandehantering är
              inte tillgänglig för alla klient eller VT om installation-
              salternativet "Endast arkiv" har valts.

              Systemet Elektronisk post gör det möjligt för användare
              att utbyta meddelanden. Användaren kan skapa, redigera,
              skriva ut, läsa, ta bort, ta emot, sända och vidare-
              befordra meddelanden. Även brevhuvudet (som bl a har
              uppgifter om mottagare och ärende) kan redigeras innan
              meddelandet sänds.

              När användaren skapar ett meddelande anger han vilka
              mottagare det skall sändas till, och eventuellt vilka
              som skall få kopior. Även mottagare av kopior och
              dolda kopior kan inkluderas. Dessutom kan användaren
              ange utgångsdatum för varje meddelande som sänds.
              Även angelägenhetsgrad (hög, låg eller normal) och
              säkerhetsgrad (t ex konfidentiellt, privat och personligt)
              kan anges. För att spara tid vid adresseringen kan

                                                                      13

 






              användaren skapa kortnamn, eller distributionslistor (som
              bara användaren själv kan se).

              Inkomna meddelanden kan vidarebefordras och/eller
              besvaras. Svaren kan sändas till avsändaren enbart eller
              till alla mottagare. Avsändaren kan begära kvittens på
              att mottagaren har fått och/eller läst ett meddelande.
              Meddelandena ges en av följande prioritetsgrader: express,
              första klass eller andra klass. Avsändaren kan själv ange
              en tidpunkt då meddelandet skall sändas.

              Meddelanden ser ut som vanliga dokument och behandlas som
              sådana av användaren. I Office Server finns mappar för
              inkommande, lästa, skapade och sända meddelanden. När ett
              meddelande har sänts har avsändare och mottagare oftast
              tillgång till samma kopia av meddelandet om de arbetar på
              samma system. Denna kopia kan inte redigeras. Meddelanden
              kan flyttas eller läggas i flera mappar i användarens
              arkiv. Även dokument från arkivet kan sändas till andra
              användare.

              Användaren kan välja att låta alla inkommande meddelanden
              vidarebefordras automatiskt till en annan användare.
              Dessutom kan användaren skapa ett svarsmeddelande som
              automatiskt sänds som svar på alla inkommande meddelanden
              t ex under semesterperioden.

              Det gemensamma arkivet i undersystemet för elektronisk
              post gör att funktionerna utförs korrekt även i gemensamma
              och distribuerade miljöer. Funktionerna i elektronisk post
              utförs alltid i lådan HUVUDLÅDA.

              En förteckning över ett meddelandes bilagor kan visas och
              bilagorna kan t ex skrivas ut eller läsas. De kan även
              sparas som redigerbar text eller som meddelanden som kan
              vidarebefordras.

              Med menyvalet Arbeta med någon annans post (AAP) kan en
              användare sköta posten för en annan användare, t ex en
              sekreterare för en chef. Menyvalet Bestäm åtkomst till
              elektronisk post (BÅE) gör det möjligt för användaren att
              ange vilka som skall få åtkomst till hans eller hennes
              post. När någon annan än ägaren skickar ett meddelande,
              visas både ägarens och avsändarens namn i brevhuvudet.

        31.1 Client Access Licensing

              Client Access Licensing har den ekonomiska fördelen att
              till en användare ge åtkomstbehörigheten till Office
              Server-systemet och DIGITAL- klienter i en enda licens.
              En CAL t ex skulle göra det möjligt för en användare
              att nå funktioner hos Office Server från en DIGITAL-
              TeamLinks-, DIGITAL- MAPI-drivrutin, webbläsare eller
              VT-klient utan att kostnaden av själva klientprogramvaran.
              Om klienten har utvecklats av en tredjepart måste dock
              rättigheterna att installera klienten på PC:n köpas från
              tredjepartsleverantören, eftersom rättigheterna till
              tredjepartsprodukter inte medges av DIGITAL Office Server
              CAL.

              Utan undantag måste en CAL finnas för varje användare i
              Office Server-profilen; dvs alla registrerade användare
              måste alltid vara licensierade av en CAL. CAL:er kan
              flyttas mellan system allteftersom användare flyttas,
              men antalet CAL:er på platsen måste vara detsamma eller

        31.4 Övergång till Office Server från MailWorks och ALL-IN-1

              MailWorks-kunder måste ersätta befintliga licenser med
              Office Server system- eller Office Server arkiv-licenser
              och skaffa en CAL eller FCCAL för varje användare på
              varje system. MailWorks-kunder på DIGITAL MDDS-kontrakt
              har fått 100 CAL:er gratis. För system med fler än 100
              användare, krävs ytterligare Uppgraderings-CAL:er (QM-
              5TD9A-C*- eller QM-5Z49A-C*-licenser om alternativet
              "Endast arkiv" har installerats) som motsvarar antalet
              abonnenter på systemet vid tiden för övergången. För
              Alpha-system tillhandahålls ytterligare uppgraderings-
              CAL:er utan kostnad. Under alla omständigheter måste
              användare som har lagts till efter övergången licensieras
              av ytterligare en CAL.

              ALL-IN-1-kunder måste behålla befintliga licenser,
              oavsett om de är av typen clusterwide, capacity eller
              user-based. De kan i stället välja alternativet "Endast
              arkiv" och de nödvändiga Arkiv Upgrade CAL:er (QM-5Z49A-
              C*) för varje abonnent. PAK:ar måste förbli installerade
              under en av Office Server V4.0A eftersom LMF-kontroller
              fortsätter. PAK:ar för sekundärt språk måste alltid förbli
              installerade.

        33.1 Begränsningar vid användning

              En TeamLinks-, MAPI Driver eller webbläsaranslutning till
              Office Server kan inte göras om Office Server-kontot är
              baserat på ett konto där flera Office Server-användare
              delar på ett enda OpenVMS-konto. Varje TeamLinks-, MAPI
              Driver eller webbläsaranvändare måste har ett individuellt
              Office Server-konto.

              Lägg även märke till:

              o  Varje Office Server-användare måste vara en registrerad
                 OpenVMS- och Office Server-användare med namnet
                 registrerat i Office Server-profilen.

              o  De som ansluter till Office Server med klienter som
                 inte levereras av DIGITAL via publicerade API:er, MAPI
                 1 Driver eller annat, måste också vara registrerade
                 OpenVMS- och Office Server-användare.
