     日本語DEC MAILworksの概要

     日本語DEC MAILworksはCCITT勧告X.400 P2で規定される国際メッセージ
     通信標準を実装した電子メッセージ通信アプリケーションです。メッセ
     ージおよび添付データを他のユーザと交換する手段を提供し，クライア
     ント／サーバの実装により異機種デスクトップ装置を通じて一貫したユ
     ーザ・インタフェースを提供します。

     日本語DEC MAILworksは，当社のVAX Massage
     Router（SPD 26.33.xx）の上に位置し，他のすべてのMAILbusゲー
     トウェイおよび補足製品と対話します。

     日本語DEC MAILworksはクライアント/サーバの実装により，ビデオ・タ
     ーミナル，DECwindows VMSワークステーションおよびその他のX11表示
     装置をサポートします。日本語DEC MAILworksは当社のVMSパーソナル・
     メール・ユーティリティ（VMSmail）や DECwindows VMSmailのユーザに
     アップグレード・パスを示しています。

     日本語DEC MAILworksはALL-IN-1統合オフィス製品とは技術的に全く依
     存していません。ALL-IN-1とは完全に独立して使用できます。

     クライアント・ソフトウェアはユーザにサービスを提供します。ソフ
     トウェアはインテリジェント・デスクトップ装置で動作し，その処理
     能力を使用します。VMSクライアント（ビデオ・ターミナルあるいは
     DECwindows）はサーバと同じノードあるいはVAXcluster上のVMSで実行
     されます。

     クライアントは，ローカルおよびワイド・エリア・ネットワークの配信
     サービスを実行する日本語DEC MAILworksサーバに接続されます。
 

     日本語DEC MAILworksサーバ間の転送や日本語DEC MAILworks
     とALL-IN-1フェーズII製品群の間の転送はVAX Message Routerによ
     り実行されます。マルチ・ベンダ環境のメール転送が必要な場合，オプ
     ションのゲートウェイをVAX Message Routerに接続することもできま
     す。

     ユーザはオフィスではインテリジェント・ワークステーションを使用
     し，自宅ではビデオ・ターミナルを使い，また旅行中はラップトップPC
     を使用できます。クライアントはこれらの状況を提供し，またサーバ上
     の同じユーザ・メールボックスやファイルを使用します。

     日本語DEC MAILworksは1984年版CCITT勧告X.400の全米標準技術研究所
     （NIST）プロファイルに準拠します。ALL-IN-1統合オフィス・システム
     の電子メッセージ通信サブシステムと互換性のある通信サービスが提供
     され，日本語DEC MAILworksで定義される新しいサービス要素が追加さ
     れています。

     日本語DEC MAILworksの機能


     VMSmailとの互換性

     ユーザはVMSmailから日本語DEC MAILworksへ容易に移行できます。イン
     タフェースはVMSmailインタフェースを論理的に拡張したものなので，
     コマンドやサービスは馴染み易くなっています。VMSmailで使用してい
     るアドレスや配布先リストは日本語DEC MAILworksでも使用することが
     できます。VMS上のクライアントからはニックネーム論理名が使用でき
     ます。また，VMSmail配布先リストで使用しているほとんどのアドレス
     は日本語DEC MAILworksでも使用できます。ただし，別のノードから配
     布先リストを使用するときと同様な制約を受けます（詳細については製
     品ドキュメントを参照）。

     ファイル・キャビネットの変換は必要ありません。

     VAX Message Router VMSmail Gatewayによって同じシステム上の，また
     は DECnetネットワーク内のVMSmailとメッセージを交換することができ
     ます。このゲートウェイは日本語DEC MAILworksワイド・エリア・ネッ
     トワーク・サーバ・パッケージに入っています。あるいは，オプション
     として購入できます。


     ユーザは日本語DEC MAILworksで使用するエディタを選択できます。
     VMSmailから呼び出したり生成できるエディタは日本語DEC MAILworksか
     らも呼び出したり生成することができます。

     ALL-IN-1との互換性

     VAX Message RouterによってALL-IN-1統合オフィス・システムと同じス
     タイルのアドレス形式でメッセージを交換することができます。

     日本語DEC MAILworksのサービスはALL-IN-1の電子メッセージ通信サブ
     システムのサービスと互換性があります。たとえば，一方の製品から
     の受信通知要求は他方でも尊重されます。CCITT勧告X.400 P2で定義さ
     れた新しい機能は，これらの2つの製品により徐々に実装されるでしょ
     う。

     ALL-IN-1統合オフィス・システムで作成された配布先リストは日本語
     DEC MAILworksでも使用できます。ただし，別のALL-IN-1オフィス・シ
     ステム・ノードにリストで送る時と同じ制約を受けます（詳細について
     は製品ドキュメントを参照）。ALL-IN-1オフィス・システムで使用され
     るアドレス指定文字列は日本語DEC MAILworksでも同様に動作します。
     ただし，ALL-IN-1の外部プロトコルを起動するアドレス文字列（下線で
     始まるもの）を除きます。

     X.400メッセージ転送

     他の日本語DEC MAILworks領域や，国際的なX.400 P2適合性基準を満た
     すサービスとメッセージを交換することができます。適合性試験はアメ
     リカ合衆国およびヨーロッパで行われています。

     日本語DEC MAILworksでは，各顧客ネットワークは"領域"と呼ばれ
     ます。顧客ネットワークの中ではDECnetが利用可能ならばどこにで
     もDECnetによってメッセージが転送されます。他ベンダのX.400サー
     ビスへの転送には OSIネットワーキング・スタックおよび日本語VAX
     Message Router X.400 Gateway（JMRX）が必要となります。

     OSIメッセージ交換には，各Digital領域(通常，顧客ネットワークに
     つき1つ)に少なくとも1つのMRX Gatewayが必要です。必要なハードウ
     ェアおよびソフトウェアについては日本語VAX Message Router X.400
     Gatewayの『ソフトウェア仕様書（SPD 26.H2.xx）』を参照してくださ
     い。

     マルチ・ベンダ環境のサポート

     MAILbusは，一連の既製ゲートウェイを提供します。また，ゲートウェ
     イ開発キットを提供し，他ベンダ所有のメール・システムとメッセージ
     を交換するためのゲートウェイを作成できます。必要なハードウェアお
     よびソフトウェアについては各製品の『ソフトウェア仕様書(SPD)』を
     参照してください。

     ディレクトリ・サービス

     日本語DEC MAILworksは，ユーザ・アドレス・ディレクトリとし
     て当社のDirectory Service（DDS）を使用します。VAX Message
     Routerの一部として利用可能なこの分散型ディレクトリは日本語DEC
     MAILworks，ALL-IN-1統合オフィス・システム，ゲートウェイおよびそ
     のディレクトリ・サービスを共有する他のエージェントとを結合しま
     す。

     ユーザはアドレス指定する際，または個人アドレス帳(PAB)に項目
     を作成する際DDSディレクトリにアクセスできます。PABは日本語DEC
     MAILworksにおいて必要不可欠であり，各ユーザの個人メール・アカウ
     ントに置かれます。ユーザは自分が割り当てたニックネームでPAB項目
     を参照します。配布先リストにはニックネームを含めることもできま
     す。

     DDSディレクトリは，サーバと接続されている間だけ利用することがで
     きるネットワーク資源です。サーバと接続していないワークステーショ
     ンでメールを作成した場合でも、ユーザはローカルにあるニックネーム
     や配布先リストを使用することができます。これらを使用しない時には
     全アドレス表記を入力してください。

     配布先リスト

     配布先リストはユーザのメール領域に格納される単純なテキスト・ファ
     イルです。この配布先リストはVMSアクセス制御機能を使用することに
     よりユーザ・グループ間で共有できます。

     ファイリング・サービス

     日本語DEC MAILworksはRMSファイルへのアクセスだけでなく，ドロア
     やフォルダといったファイリング構造も提供します。ファイリング・
     サービスは事実上VMSmailと似ています。ローカルに記憶装置を持った
     DECwindows装置では，サーバにドロアを置く以外にローカルにもドロア
     を置くことができます。
     VMSのドロアは, VMSアクセス制御で指定された各ユーザの特権によりユ
     ーザ・グループ間で共有できます。

     メッセージ検索

     ユーザが指定したヘッダフィールドおよびファイリング構造に基づい
     て，メッセージを検索することができます。

     ファイル形式

     後述するように，複数のエディタが様々なインタフェースで使用できま
     す。ただし，DECwindows装置に適したエディタはビデオ・ターミナルで
     は適さない場合があります。多様なファイル形式がこれらのエディタに
     よって生成されます。日本語DEC MAILworksは，様々なファイル形式を
     処理できるように設計されており，メッセージの読み易さを最大限にす
     るように構成できます。ASCII，DDIF，WPS-PLUSおよびDXで書かれたメ
     ッセージを自動処理したり，当社のコンパウンド・ドキュメント・アー
     キテクチャ(CDA)に準拠した他のファイル書式化ルーチンや変換ルーチ
     ンを起動することができます。

     クライアントは起動時に，添付文書のファイル形式に適合したビューア
     やアプリケーションを起動することができます。

     日本語DEC MAILworksではメッセージが配信されたときに，その構成
     要素が自動的に変換されるように設定できます。これには当社のCDA
     Converter Library for VMS（SPD 31.31.xx）のようなオプションのVMS
     常駐型変換サービスまたはKEYpak[R] for VMS（SPD 32.71.xx）のよう
     なサード・パーティのコンバータを使用します。これは別料金のオプシ
     ョンです。ユーザは好みのファイル形式をプロファイルに登録できま
     す。

     特定のファイル形式を交換する場合，特にゲートウェイを経由する場合
     は，最寄りの日本DECの各支店/営業所にお問い合わせください。

     DDIFドキュメントをALL-IN-1バージョン2.3と交換するのが難しい場合
     があります。詳細については最寄りの日本DECの各支店/営業所にお問い
     合わせください。


     ヘルプ

     ユーザはHELP機能を使用して日本語DEC MAILworksのヘルプを見ること
     ができます。HELPはユーザがドキュメントを参照しなくても，当該機能
     の実行方法など基本的な情報を提供します。

     印刷

     日本語DEC MAILworksはVMSで提供される当社の標準VMS印刷機能を利用
     します。ユーザはプリンタの出力先およびオプションを指定できます。

     メッセージ通信機能

     ユーザは提供されているコマンドを使用して，メッセージを作成したり
     読んだり送ることができます。メッセージには，当社の複合文書用の文
     書交換形式（DDIF）を含む多様な種類のファイル形式で情報を入れるこ
     とができます。

     メッセージは別の宛先に回送できます。また，メッセージの添付文書に
     は別のメッセージ，ドキュメント，ファイルあるいはバイナリ・ファイ
     ルでさえ入れることができます。

     メッセージの応答は簡単です。発信者へ送り返す応答は自動的にアドレ
     ス指定されます。ユーザは送信者，受信者全員，または返信先リストに
     返信することができます。

     メッセージは有効なMAILbusの宛先（たとえばVMSmail，ULTRIXmailまた
     はX.400など）ならば世界中の一人ないし複数のユーザにアドレス指定
     できます。ファイル形式は受信者がそのメモを読めるように，ゲートウ
     ェイまたはユーザ・エージェントによって変更される場合があります。
     全てのファイル形式が全てのゲートウェイおよびユーザ・エージェント
     で処理されるわけではありません。メモの読み易さを最大限に保証する
     には，ネットワーク能力の計画に配慮が必要です。

     通知

     配信および受信タグをメッセージに付けることができます。このタグに
     よってメッセージが配信または受信されたときに通知メッセージが生成
     され発信者へ返されます。
 

     新しいメールの通知

     新しいメッセージの到着の通知がオンラインで表示されます。

     メッセージ宛先変更

     ユーザは，すべての受信メッセージを自動的に第2のアドレスへ宛先変
     更するように要求できます。

     優先度に基づく配信

     ユーザは，メッセージ配信の優先度を選択できます。配信の等級（緊
     急，普通，不急）を指定します。その他の転送サービスも利用できま
     す。

     メッセージ・クラス

     ユーザは複数の指標を設定し，メッセージの性質について受信者に情報
     を与えることができます。指標には重要度（至急，普通，低い）と秘密
     度（制限なし，私信，親展，社外秘）があります。

     失効日をメッセージに付けることができ，超過したメッセージの削除に
     役立ちます。

     これらのフィールドはファイル・キャビネット中のメッセージの分類に
     も使用できます。

     システム管理

     メール・システムのローカル・システム管理は最低限必要ですが，通常
     は自動化されています。標準VMSアカウントを開設するにはシステム管
     理者または監督者を必要としますが，大部分の業務はユーザが実行でき
     ます。

     ファイル・キャビネットの保守用に実行する特別なユーティリティはあ
     りません。サーバ上の1つあるいはすべてのユーザ領域は，標準VMSユー
     ティリティを使用して，保存したり回復することができます。

     メール管理設計はMAILbus製品群で使用される統合管理システムに従い
     ます。日本語DEC MAILworksサーバは，管理ユーティリティを提供する
     ので構成を変更したり，メール・システムの状態を調べることができま
     す。VMS SYSMANユーティリティを介してこのユーティリティを起動する
     と，上記のような操作が集中管理して行えます。

     日本語DEC MAILworksドキュメント

     インストレーション，システム管理およびユーザ・プロシージャに関す
     るドキュメントは個々のキットに含まれています。サーバ媒体キットに
     はビデオ・ターミナルおよびVMS DECwindowsクライアントのソフトウェ
     アとインストレーション説明書が含まれています。

     パッケージング

     日本語DEC MAILworksビデオ・ターミナル クライアント，日本語DEC
     MAILworks VMS DECwindowsクライアントおよび日本語DEC MAILworks
     VMSサーバは VMS上のレイヤード・ソフトウェア製品です。

     日本語DEC MAILworksビデオ・ターミナル クライアント

     日本語DEC MAILworksビデオ・ターミナル クライアントはVMSmailに似
     ていますが，Message Routerと直接通信し，メッセージ通信サービスを
     広範囲にわたってアクセスする豊富なプロトコルを使用しています。こ
     のクライアントは日本語DEC MAILworksサーバと同じところに置かれま
     す。通常の端末から日本語DEC MAILworksのサービスに一様にアクセス
     できます。

     このクライアントは，他のクライアントと同じサーバを共有するので，
     デスクトップ装置によって，ユーザは家で仕事をすることができます。

     VMSmailとの互換性

     ユーザはVMSパーソナル・メール・ユーティリティ（VMSmail）から日本
     語DEC MAILworksビデオ・ターミナル クライアントに移行することがで
     きます。コマンド行インタフェースはVMSmailコマンド・インタフェー
     スを論理的に拡張したものなので，日本語DEC MAILworksビデオ・ター
     ミナル クライアントのコマンドおよびサービスには親しみ易いでしょ
     う。VMSmailで使用しているアドレス，ニックネーム論理名および配布
     先リストは日本語DEC MAILworksビデオ・ターミナル クライアントでも
     使用できます。

     日本語DEC MAILworksビデオ・ターミナル クライアントでは既存の
     VMSmailファイル・キャビネットのメッセージを指定したり見ることが
     できます。また，日本語DEC MAILworksファイル・キャビネットにそれ
     らを複写することもできます。

     エディタ

     省略時のエディタは簡単なテキスト・エディタです。

     VMSを使用しているユーザは，サード・パーティのエディタと同様に当
     社の複数のエディタの中から選択できます。VMSmailから呼び出したり
     生成できるエディタならば，日本語DEC MAILworksからでも呼び出した
     り生成したりすることができます。

     日本語DEC MAILworks VMS DECwindowsクライアント

     日本語DEC MAILworks VMS DECwindowsクライアントは日本語DEC
     MAILworksのクライアント・ソフトウェアをDECwindows上に実装し
     たものです。このクライアントはDECwindowsスタイル・ガイドに準拠し
     ており，サーバの動作しているのと同じローカル・エリアVAXクラスタ
     のメンバのワークステーション上で動作します。また，表示を当社のサ
     ポートするX11表示装置上にすることもできます。

     他のプラットフォームからのアクセス

     日本語DEC MAILworksはDECwindowsの標準に準拠しているので，X-サー
     バの動作する装置を持っているユーザはメール・ソフトウェアを使用す
     ることができます。この場合，メッセージの作成や保存はデスクトップ
     装置ではなく，日本語DEC MAILworksが行います。ウィンドウの表示は
     デスクトップ装置により行われます。これにより，DECwindows標準に準
     拠しているデスクトップ装置にアクセスできます。

     VMSmailおよびDECwindows VMSmailとの互換性

     ユーザはVMSパーソナル・メール・ユーティリティ（VMSmail）や
     DECwindows VMSmailから日本語DEC MAILworks VMS DECwindowsクラ
     イアントに移行することができます。インタフェースはDECwindows
     VMSmailに似ているので，日本語DEC MAILworks VMS DECwindowsクライ
     アントのコマンドおよびサービスには親しみやすいでしょう。VMSmail
     で使用しているアドレス，ニックネーム論理名および配布先リストは日
     本語DEC MAILworks VMS DECwindowsクライアントでも使用できます。

     日本語DEC MAILworks VMS DECwindowsクライアントはメッセージを既存
     のVMSmailのファイル・キャビネットから日本語DEC MAILworksのファイ
     ル・キャビネットに複写することができます。

     エディタ

     省略時のエディタは簡単なテキスト・エディタです。

     DECwindowsのユーザは，当社の複数のエディタの中から選択できま
     す。また，サード・パーティのエディタも使用できます。DECwindows
     VMSmailから呼び出したり生成できるエディタならば，日本語DEC
     MAILworks DECwindows クライアントからでも呼び出したり生成し
     たりすることができます。エディタとして DECwriteを使用することも
     できます。

     必要なハードウェア

     『システム・サポート付加情報(SSA 27.P7.00-x)』に記載されている
     VAX，MicroVAX，VAXserverおよび
     VAXstationなどの構成が必要です。

     必要なソフトウェア

     クライアントをサポートする日本語DEC MAILworks VMSサーバおよび日
     本語DEC MAILworksビデオ・ターミナル クライアントには，以下のソフ
     トウェアが必要です。

     o  日本語VMSオペレーティング・システム

     o  DECnet-VAX

     日本語DEC MAILworksサーバは，同じサーバを使用する日本語DEC
     MAILworksクライアントのユーザにメッセージを配信します。した
     がって，単一VAXclusterまたは単一LANの全ユーザは，1つの日本語DEC
     MAILworksサーバを介してメッセージを交換することができます。

     VAX Message Routerは他のサーバのユーザまたは他の電子メッセージ通
     信製品のユーザとメッセージを交換するために必要です。VMSmailユー
     ザと通信するには，さらにVMSmail Gatewayが必要です。

     ゲートウェイ

     Message RouterはVMSmail Gatewayと共にパッケージされているので，
     次のパッケージのうち一つを選択してください。

     o  VAX Message Router

     あるいは

     o  VAX Message Router VMSmail Gateway

     大部分の構成でMessage Routerが必要です。また，VMSmail製品から
