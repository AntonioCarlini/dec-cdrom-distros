DESCRIPCION

DECdecision es una aplicación basada en VMS y estaciones de trabajo que
funciona en el entorno de equipos DECwindows de oficina y que sirve de 
apoyo en la toma de decisiones.

Consta de cuatro componentes de software principales: Access 
para consultas ad-hoc a bases de datos, Calc para analizar datos 
comerciales y financieros, Chart para generar gráficos comerciales y 
Builder para registrar y automatizar tareas repetitivas multi-componente.

DECdecision funciona en el entorno X-Window de DECwindows y tiene su 
interfaz y funcionamiento habitual de menús desplegables, áreas de diálogo, 
ventanas, operaciones de cortar y pegar en los componentes, y dispositivo 
de señalización de ratón. Por lo tanto, DECdecision tiene un 
interfaz consistente en todo el producto, compartido con otras aplicaciones 
(tales como DECwrite) desarrolladas con DECwindows.

Características

Access de DECdecision (base de datos)

Access es el componente de gestión de bases de datos de DECdecision y 
proporciona acceso distribuido a datos locales y remotos, un enfoque 
simplificado de ventanas para manipulación y consulta de datos, y creación 
de bases de datos personales.

Access consta de:

o   Funciones de manipulación de datos que incluyen "Ver menos" para crear 
    subconjuntos de grupos de datos, "Ver más" para ampliar un conjunto, 
    "Ver una" para ver sólo una aparición de un valor, etc. y "Clasificar" 
    para reorganizar los datos en orden ascendente o descendente por uno o 
    más campos.

o   Edición de datos, incluyendo una fácil introducción de datos, 
    actualización y borrado, cortado y pegado, anulación de la última entrada 
    tecleada, etc.

o   Cálculo de funciones que incluyen Total (Suma), Media, Máximos y
    Mínimos por subgrupos de registros y columnas calculadas donde una 
    columna se basa en una o más columnas o en una columna 
    multiplicada por un valor constante, etc.

o   Carpetas públicas para compartir bases de datos dentro de un equipo de 
    trabajo y carpetas personales para bases de datos personales.

o   Opciones para protección de datos que incluyen "Protección" para activar o 
    restringir el número de usuarios que pueden leer o modificar las tablas 
    y "Modo de Acceso" para determinar la forma en que los usuarios pueden 
    abrir las tablas (Sólo lectura o Actualización).

o   Utilidad Trazar gráfico para llamar y pasar datos a DECchart 
    automáticamente.
    
o   Registro y reproducción para capturar instrucciones para su uso 
    posterior.

o   Utilidad Ver documento para almacenar y visualizar una referencia 
    de la base de datos en un documento DDIF tal como a una carta 
    creada con DECwrite o a una imagen procesada con "Scanner" que se haya 
    convertido a DDIF.

o   Función Unir para combinar otras tablas de DECdecision, o distintos 
    de ficheros de datos de otras fuentes que estén admitidas.

o   Utilidad Referencia para crear y mantener referencias a datos fuera de 
    Access. Incluye: bases de de datos Rdb/VMS, dominios DATATRIEVE, ficheros 
    RMS y base de datos DBMS a través de DATATRIEVE, y ficheros IBMRO 
    IDMS/RTM y DB2 a través de VIDA. (Los ficheros RMS, bases de datos DBMS, 
    y datos IBM son sólo de lectura.)

o   Generación de informes sencilla utilizando datos de fuentes
    únicas o múltiples. Ofrece al usuario la posibilidad de hacer 
    subconjuntos y clasificar datos, fijar saltos de línea y de página y 
    salvar o imprimir el resultado en formatos ASCII, PostScript (R) o DDIF. 
    Los informes DDIF se utilizan en DECwrite.

Calc de DECdecision

Calc es el componente analítico principal de DECdecision que combina su
interfaz de hojas de cálculo en ventana para facilitar la construcción de 
expresiones y fórmulas, un lenguaje de macros de hojas de cálculo, y 
funciones matemáticas, estadísticas y financieras.

Calc consta de:

o   Funciones financieras, que incluyen valor futuro (VF), tasa interna de
    rendimiento (TRI), valor actual neto (VA), análisis de restitución de 
    inversiones, tasas de intereses, pagos, etc...

o   Funciones matemáticas tales como valores absolutos, exponenciación,
    logaritmos (de base 10 y E), raíces cuadradas y redondeo.
             
o   Funciones estadísticas para calcular medias, conteos, máximos, mínimos, 
    varianzas, desviaciones típicas y sumas.

o   Funciones trigonométricas para transformaciones y cálculo de senos, 
    cosenos y tangentes.
          
o   Operadores booleanos que incluyen Y, O, N, y condiciones.

o   Valores constantes que incluyen E (2,71828), PI (3,14159), Si para 
    condiciones y ND (no disponible) para valores que falten.

o   Funciones fecha/hora incluyendo funciones de calendario, semanas, años, 
    fecha, fecha y hora, etc.

o   Funciones de gráficos para actualización dinámica de gráficos. Al 
    cambiar los valores de las celdas el gráfico se actualiza automáticamente.

o   Posibilidades de enlace y consolidación para combinar varias hojas de 
    cálculo (locales y remotas) en una hoja principal.

o   Posibilidad de ocultar y bloquear datos para proteger aquellos que sean 
    confidenciales.

o   Registro y reproducción para capturar instrucciones de hojas de cálculo 
    para su uso posterior.

o   Una tabla de hipótesis para realizar análisis con diferentes valores 
    utilizando las mismas fórmulas.

o   Un lenguage interno de programación de procedimientos para usuarios 
    avanzados que desarrollen macros complejas.

DECchart (Gráficos)

Chart es el componente gráfico integrado en DECdecision que trabaja 
directamente con datos de Calc, de Access y con ficheros de fuentes externas 
(DECgraph, ASCII y Lotus WK1). Chart tiene un conjunto de estilos
que ofrece al usuario una serie de gráficos predefinidos que 
pueden ser personalizados a través de leyendas, cuadrículas, títulos, etc. 
Ofrece una amplia variedad de 45 estilos de gráficos basados en los 
siguientes estilos básicos: líneas, área, barras, columnas, 
histogramas, sectores, dispersión, fluctuación y combinaciones.

Chart consta de:

o   Anotación gráfica, complementos: texto, líneas, rectángulos y cabezas 
    de flecha en cualquier punto del gráfico para resaltar algo concreto y 
    personalizarlo.

o   Color en pantalla, asigna atributos de color a: ejes, renglones, 
    marco, cuadrícula, leyendas, etiquetas, marcas, columnas y relleno de 
    barras y objetos complementarios.

o   Modificación fácil de atributos de gráfico incluyendo: ejes, 
    renglones, valores de datos, marco, cuadrícula, leyenda, etiquetas, 
    marca, sombreados, espaciado y superposición de barras.

o   Flexibilidad de ficheros: salvar datos, estilo de gráfico, complementos 
    de gráfico y selección de colores, por separado o todo junto.

o   Posibilidad de elegir tramas de relleno, tramas de líneas, tipos de 
    marcas y atributos de color.

o   Interfaz de hoja de trabajo para hipótesis gráficas fácil de utilizar.

o   Modificación fácil de atributos de texto. Se puede cambiar: tamaño de 
    letra, familia, intensidad e inclinación.

Builder de DECdecision (herramienta de automatización de procedimientos)

Builder es la herramienta de automatización multi-componente de DECdecision 
que construye procedimientos para registrar operaciones multi-ventana
(por ejemplo, llamada a aplicaciones, flujo de datos y control 
datos) entre componentes para producir procedimientos automatizados. 
Builder tiene la posibilidad de registrar, almacenar y reproducir 
procedimientos que se repitan con frecuencia y que realicen componentes 
integrados en DECdecision.

Consta de:
             
o   Modo de registro automático para capturar flujo de información y 
    secuencias de instrucciones automáticamente.

o   Modo de registro manual para elegir herramientas, aplicaciones no 
    integradas y flechas para flujos de datos.

o   Modo de reproducción para ejecutar el procedimiento de Builder 
    interactivamente o en modo secundario.

o   Modo de depuración para identificar errores y corregirlos en los 
    procedimientos construidos con Builder.

o   Manual para almacenar y leer la descripción de un procedimiento.

o   Caja de herramientas para almacenar conexiones a aplicaciones no 
    integradas tales como MAIL. Una aplicación no integrada es cualquier 
    aplicación o programa que se pueda llamar desde DCL. Builder no puede 
    registrar automáticamente aplicaciones no integradas.

o   Edición de características como cortar, pegar, duplicar y borrar.
          
o   Posibilidades de crear procedimientos integrados dentro de otros 
    procedimientos de Builder y de integrar procedimientos de Access y de Calc.

Características de todo el sistema

Facilidad de uso

DECdecision ofrece un panel de control para un arranque adecuado de los 
componentes, personalización para modificar varias características del 
entorno de DECdecision y ayuda para una introducción general a cada 
componente. Además, DECdecision dispone de ventanas con menús contextuales 
en todo el producto para ver rápidamente una lista de las funciones 
disponibles y de aceleradores y línea de instrucciones para usuarios con 
experiencia.

AYUDA

DECdecision ofrece al usuario dos formas de llamar a la ayuda. Desde el 
panel de control, los usuarios pueden elegir "Introducción" para recibir 
una explicación de cada componente. En cualquier momento, en DECdecision se 
puede llamar a la ayuda para disponer de una explicación detallada de una 
función concreta.

Relación con DECwrite

DECwrite es una aplicación de tratamiento de documentos compuestos que 
permite a los usuarios de VMS crear y dar formato a documentos que incluyan 
texto, gráficos, imágenes y datos de aplicaciones admitidas.

Los informes generados dentro de Access, Calc y gráficos de Chart se pueden 
incorporar a DECwrite a través de la utilidad de enlaces activos de 
DECwrite. Además, DECwrite puede leer directamente los ficheros de informes 
de DECdecision con formato DDIF.
