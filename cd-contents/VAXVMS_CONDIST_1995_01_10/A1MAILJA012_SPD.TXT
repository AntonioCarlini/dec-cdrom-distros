1  日本語DEC MAILworksの概要

本ソフトウェア仕様書では，日本語DEC MAILworks製品群の3つの構成要素である，日
本語DEC MAILworks for OpenVMSサーバ，日本語DEC MAILworks for OpenVMSビデオ・
ターミナル，日本語DEC MAILworks for OpenVMS DECwindowsについて説明します。

日本語DEC MAILworksはCCITT勧告X.400 P2で規定される国際メッセージ通信標準を実
装した電子メッセージ通信アプリケーションです。メッセージおよび添付データを他
のユーザと交換する手段を提供し，クライアント／サーバの実装により異機種デスク
トップ装置を通じて一貫したユーザ・インタフェースを提供します。

日本語DEC MAILworksは，当社のVAX Massage
Router（SPD 26.33.xx）の上に位置し，他のすべてのMAILbusゲートウェイおよび
補足製品と対話します。

日本語DEC MAILworksはクライアント/サーバの実装により，ビデオ・ターミナル，
Apple[R] Macintosh[R]，DECwindows OpenVMSワークステーションおよびその他のX11
表示装置をサポートします。日本語DEC MAILworksは当社のOpenVMSパーソナル・メー
ル・ユーティリティ（VMSmail）や DECwindows VMSmailのユーザにアップグレード・
パスを示しています。

日本語DEC MAILworksはALL-IN-1統合オフィス製品には技術的に全く依存していませ
ん。ALL-IN-1とは完全に独立して使用できます。

クライアント・ソフトウェアはユーザにサービスを提供します。ソフトウェアはイン
テリジェント・デスクトップ装置で動作し，その処理能力を使用します。VMSクライ
アント（ビデオ・ターミナルあるいはDECwindows）はサーバと同じノードあるいは
VAXcluster上のOpenVMSで実行されます。

クライアントは，ローカルおよびワイド・エリア・ネットワークの配信サービスを実
行する日本語DEC MAILworksサーバに接続されます。

1.1  日本語DEC MAILworksの機能


■転送

日本語DEC MAILworks for OpenVMSは，クライアントとサーバ間の転送接続をする
DECnet，TCP/IP，またKermitを使用した非同期接続の3つをサポートしています。

日本語DEC MAILworksはこれらすべての転送を同時にサポートします。サーバに接続す
る際にどの転送経路を使用するかはクライアント側で指定します。省略時設定では，
サーバにはDECnetと非同期接続がサポートされています。TCP/IP接続を使用する場
合，転送が可能になるように構成を適切に変更するとともに，適切なソフトウェアが
サーバ上で実行されている必要があります。

クライアントからDEC MAILworksサーバへの接続にDECnetが使用されていなくても，
DEC MAILworksサーバからVAX Message Router Transfer Service，VAX Message
Router Directory Service (DDS)への様々な接続をするためにDECnetがVAX上で実
行される必要があります。

■VMSmailとの互換性

ユーザはVMSmailから日本語DEC MAILworksへ容易に移行できます。インタフェースは
VMSmailインタフェースを論理的に拡張したものなので，コマンドやサービスは馴染
み易くなっています。VMSmailで使用しているアドレスや配布先リストは日本語DEC
MAILworksでも使用することができます。OpenVMS上のクライアントからはニックネー
ム論理名が使用できます。また，VMSmail配布先リストで使用しているほとんどのア
ドレスは日本語DEC MAILworksでも使用できます。ただし，別のノードから配布先リ
ストを使用するときと同様な制約を受けます（詳細については製品ドキュメントを参
照）。

ファイル・キャビネットの変換は必要ありません。

VAX Message Router VMSmail Gatewayによって同じシステム上の，または DECnetネッ
トワーク内のVMSmailとメッセージを交換することができます。このゲートウェイは日
本語DEC MAILworksワイド・エリア・ネットワーク・サーバ・パッケージに入っていま
す。あるいは，オプションとして購入できます。

ユーザは日本語DEC MAILworksで使用するエディタを選択できます。VMSmailから呼び
出したり生成できるエディタは日本語DEC MAILworksからも呼び出したり生成すること
ができます。

■ALL-IN-1との互換性

VAX Message RouterによってALL-IN-1統合オフィス・システムと同じスタイルのアド
レス形式でメッセージを交換することができます。

日本語DEC MAILworksのサービスはALL-IN-1の電子メッセージ通信サブシステムのサー
ビスと互換性があります。たとえば，一方の製品からの受信通知要求は他方でも尊重
されます。CCITT勧告X.400 P2で定義された新しい機能は，これらの2つの製品により
徐々に実装されるでしょう。

ALL-IN-1統合オフィス・システムで作成された配布先リストは日本語DEC MAILworksで
も使用できます。ただし，別のALL-IN-1オフィス・システム・ノードにリストで送る
時と同じ制約を受けます（詳細については製品ドキュメントを参照）。ALL-IN-1オフ
ィス・システムで使用されるアドレス指定文字列は日本語DEC MAILworksでも同様に動
作します。ただし，ALL-IN-1の外部プロトコルを起動するアドレス文字列（下線で始
まるもの）を除きます。

■X.400メッセージ転送

他の日本語DEC MAILworks領域や，国際的なX.400 P2適合性基準を満たすサービスとメ
ッセージを交換することができます。

日本語DEC MAILworksでは，各顧客ネットワークは"領域"と呼ばれます。顧客ネットワ
ークの中ではDECnetが利用可能ならばどこにでもDECnetによってメッセージが転送さ
れます。他ベンダのX.400サービスへの転送には OSIネットワーキング・スタックおよ
び日本語VAX Message Router X.400 Gateway（JMRX）が必要となります。

OSIメッセージ交換には，各Digital領域(通常，顧客ネットワークにつき1つ)に少
なくとも1つのMRX Gatewayが必要です。必要なハードウェアおよびソフトウェアに
ついては日本語VAX Message Router X.400 Gatewayの『ソフトウェア仕様書（SPD
26.H2.xx）』を参照してください。

■マルチ・ベンダ環境のサポート

MAILbusは，一連の既製ゲートウェイを提供します。また，ゲートウェイ開発キットを
提供し，他ベンダ所有のメール・システムとメッセージを交換するためのゲートウェ
イを作成できます。必要なハードウェアおよびソフトウェアについては各製品の『ソ
フトウェア仕様書(SPD)』を参照してください。

■ディレクトリ・サービス

日本語DEC MAILworksは，ユーザ・アドレス・ディレクトリとして当社のDirectory
Service（DDS）を使用します。VAX Message Routerの一部として利用可能なこの分散
型ディレクトリは日本語DEC MAILworks，Team Links Mail，ALL-IN-1統合オフィス・
システム，ゲートウェイおよびそのディレクトリ・サービスを共有する他のエージェ
ントとを結合します。

また，日本語DEC MAILworksクライアントは個人アドレス帳(PAB)を提供します。ユー
ザは，PABを利用して，名前，電話番号，ニック・ネームなどに従ってメール・アドレ
スを保管，検索することができます。

他にも日本語DEC MAILworks OpenVMSクライアントでは，複数のユーザにメールを送付
する場合に便利な配布リストを作成したりアクセスすることができます。これらの配
布リストはOpenVMSアクセス制御機能を利用して，他のユーザと共有することもできま
す。

宛名書きをしながら，DDS，PAB，配布リストにアクセスすることもできます。これら3
つのサービスは，メッセージ作成中に合わせて利用します。

■ファイリング・サービス

日本語DEC MAILworksはRMSファイルへのアクセスだけでなく，ドロアやフォルダとい
ったファイリング構造も提供します。ファイリング・サービスは事実上VMSmailと似て
います。ローカルに記憶装置を持ったDECwindows装置では，サーバにドロアを置く以
外にローカルにもドロアを置くことができます。

OpenVMSのドロアは, OpenVMSアクセス制御で指定された各ユーザの特権によりユー
ザ・グループ間で共有できます。

■メッセージ検索

ユーザが指定したヘッダフィールドおよびファイリング構造に基づいて，メッセージ
を検索することができます。

■ファイル形式

後述するように，複数のエディタが様々なインタフェースで使用できます。ただ
し，DECwindows装置に適したエディタはビデオ・ターミナルでは適さない場合があ
ります。多様なファイル形式がこれらのエディタによって生成されます。日本語DEC
MAILworksは，様々なファイル形式を処理できるように設計されており，メッセージの
読み易さを最大限にするように構成できます。ASCII，DDIF，WPS-PLUSおよびDXで書か
れたメッセージを自動処理したり，当社のコンパウンド・ドキュメント・アーキテク
チャ(CDA)に準拠した他のファイル書式化ルーチンや変換ルーチンを起動することがで
きます。

クライアントは起動時に，添付文書のファイル形式に適合したビューアやアプリケー
ションを起動することができます。

日本語DEC MAILworksではメッセージが配信されたときに，その構成要素が自動的
に変換されるように設定できます。これには当社のCDA Converter Library for VMS
（SPD 31.31.xx）のようなオプションのVMS常駐型変換サービスまたはKEYpak[R] for
OpenVMS（SPD 32.71.xx）のようなサード・パーティのコンバータを使用します。これ
は別料金のオプションです。ユーザは好みのファイル形式をプロファイルに登録でき
ます。

特定のファイル形式を交換する場合，特にゲートウェイを経由する場合は，最寄りの
日本DECの各支店/営業所にお問い合わせください。

■ヘルプ

ユーザはHELP機能を使用して日本語DEC MAILworksのヘルプを見ることができます。
HELPはユーザがドキュメントを参照しなくても，当該機能の実行方法など基本的な情
報を提供します。

■印刷

日本語DEC MAILworksはOpenVMSで提供される当社の標準OpenVMS印刷機能を利用しま
す。ユーザはプリンタの出力先およびオプションを指定できます。

■メッセージ通信機能

ユーザは提供されているコマンドを使用して，メッセージを作成したり読んだり送る
ことができます。メッセージには，当社の複合文書用の文書交換形式（DDIF）を含む
多様な種類のファイル形式で情報を入れることができます。

メッセージは別の宛先に回送できます。また，メッセージの添付文書には別のメッセ
ージ，ドキュメント，ファイルあるいはバイナリ・ファイルでさえ入れることができ
ます。

メッセージの応答は簡単です。発信者へ送り返す応答は自動的にアドレス指定されま
す。ユーザは送信者，受信者全員，または返信先リストに返信することができます。

メッセージは有効なMAILbusの宛先（たとえばVMSmail，ULTRIXmailまたはX.400など）
ならば世界中の一人ないし複数のユーザにアドレス指定できます。ファイル形式は受
信者がそのメモを読めるように，ゲートウェイまたはユーザ・エージェントによって
変更される場合があります。全てのファイル形式が全てのゲートウェイおよびユー
ザ・エージェントで処理されるわけではありません。メモの読み易さを最大限に保証
するには，ネットワーク能力の計画に配慮が必要です。

■通知

配信および受信タグをメッセージに付けることができます。このタグによってメッセ
ージが配信または受信されたときに通知メッセージが生成され発信者へ返されます。

■新しいメールの通知

新しいメッセージの到着の通知がオンラインで表示されます。

これらのフィールドはファイル・キャビネット中のメッセージの分類にも使用できま

ています。標準OpenVMSアカウントを開設するにはシステム管理者または監督者を必要

ファイル・キャビネットの保守用に実行する特別なユーティリティはありません。サ

メール管理設計はMAILbus製品群で使用される統合管理システムに従います。日本語

ル・システムの状態を調べることができます。OpenVMS SYSMANユーティリティを介し
てこのユーティリティを起動すると，上記のような操作が集中管理して行えます。

■日本語DEC MAILworksドキュメント

インストレーション，システム管理およびユーザ・プロシージャに関するドキュメン
トは個々のキットに含まれています。サーバ媒体キットにはビデオ・ターミナルおよ

1.3  日本語DEC MAILworks OpenVMS DECwindowsクライアント

日本語DEC MAILworks OpenVMS DECwindowsクライアントは日本語DEC MAILworksのク

■他のプラットフォームからのアクセス

日本語DEC MAILworksはDECwindowsの標準に準拠しているので，X-サーバの動作する

合，メッセージの作成や保存はデスクトップ装置ではなく，日本語DEC MAILworksが
行います。ウィンドウの表示はデスクトップ装置により行われます。これにより，
DECwindows標準に準拠しているデスクトップ装置にアクセスできます。

MAILworks OpenVMS DECwindowsクライアントのコマンドおよびサービスには親しみや
すいでしょう。VMSmailで使用しているアドレス，ニックネーム論理名および配布先リ

です。しかし，VAX Message Router Directory Service (DDS)は日本語DEC MAILworks
サーバと同一システムに必要です。VMSmailユーザと通信するには，さらにVMSmail
Gatewayが必要です。
